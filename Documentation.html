WEB API

Based on simple REST principles, the Web API endpoints return JSON metadata about the recource you're looking for (if we have it of course).

Requests

Method Action
GET Retrieves resources
POST Creates resources
PUT Replaces or updates resources
DELETE Deletes resources

Response Status Code

Web API uses the following response status codes.

Status Code Description
200 OK- the request has succeeded. The client can read the result of the request in the
body and the headers of the response.
400 Bad Request - The request could not be understood by the server due to
malformed syntax.
403 Forbidden - The client does not have access rights to the contents.
404 Not Found -
The server cannot find the requested resource.

Functions

Function Key Rest Element
The end point of the URI: /api/contents The endpoint is a resource Identifier, which specifies the specific location the resource the user is trying to acess
HTTP methods: GET,POST,DELETE,PUT (CRUD) These methods are used for pure rest implementation, specifically to retrieve a resource or information about a resource
The MIME format (JSON) Java script object notation(JSON) is used to translate code into a human readable language. This is used for the developer to properly communicate with their API.
